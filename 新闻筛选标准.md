# 财经日报雷达新闻筛选标准

本文档用于指导 RSS/网页抓取后的新闻准入、过滤、评分、入库与日报生成。目标不是“抓得越多越好”，而是减少生活方式、泛科技、社会新闻、观点散文等干扰项，让最终日报只保留对财经科技判断有增量的信息。

## 1. 总体原则

### 1.1 日报定位

本项目的日报服务于“财经科技研究判断”，优先覆盖以下六大赛道：

1. AI 应用 / AI Agent
2. 半导体 / PCB
3. 具身智能与机器人
4. IPO / A股科技
5. 港股科技
6. 全球市场与宏观变量

任何新闻进入最终日报，至少要满足以下条件之一：

- 属于上述六大赛道之一，并且包含明确事件、主体或数据。
- 虽不属于六大赛道，但对 A股、港股、美股、利率、汇率、商品、科技产业链或风险偏好有直接影响。
- 来自官方、交易所、监管机构、上市公司公告、国家统计局等高权威来源，且内容为政策、数据、监管、公告或市场指标。

### 1.2 三层过滤

建议不要只做关键词命中，而是做三层过滤：

1. **采集前源头约束**：不同 RSS 源设定不同抓取上限、栏目白名单、源权重。
2. **入库前硬过滤**：删除明显非财经、非科技、非市场相关内容。
3. **日报前评分排序**：对剩余内容打分，只把高分和必要候选纳入日报。

## 2. 时间窗口

### 2.1 默认窗口

- 日报主窗口：过去 24 小时。
- 周末或节假日：可放宽到过去 72 小时，但需要标记“补充观察”。
- 历史深度文章：只有在解释当日重大事件时才保留，且不能挤占主新闻位置。

### 2.2 时间过滤规则

保留：

- 发布时间在主窗口内。
- 官方数据或公告虽发布时间略早，但市场影响在当天发酵。
- 同一事件在当天有新进展、新监管动作、新价格反应或新公告。

剔除：

- 过期转载、回顾文章、无新增信息的旧闻重发。
- RSS 中没有发布时间且无法从页面解析时间的文章，除非来源权威且标题高度相关。

## 3. 硬过滤标准

以下内容即使来自已订阅 RSS，也默认不进入日报。

### 3.1 明显排除主题

排除：

- 体育、赛事、球迷、世界杯、俱乐部、运动员，除非涉及上市公司并购、赞助金额、博彩监管或体育产业上市公司重大财务影响。
- 娱乐、影视、明星、综艺、网红、游戏八卦，除非涉及上市公司重大财报、监管处罚、并购或商业模式变化。
- 生活方式、餐饮、旅游、城市文化、书店、教育情怀、职场鸡汤、消费体验类文章。
- 社会民生、食品安全、交通事故、治安案件、校园事件，除非影响重点上市公司、监管政策或行业供需。
- 纯观点、散文、人物故事、商业史、创业复盘，除非当天触发资本市场或产业链重大变化。
- 纯营销稿、品牌软文、产品种草、活动预告、会议招商。
- 只讨论国外社会政治冲突但没有明确市场传导链条的国际新闻。

### 3.2 低信息量标题

排除：

- “一文看懂”“为什么说”“谁在买单”“XX 怎么了”这类泛分析标题，除非正文或摘要出现明确公司、金额、政策、数据或交易动作。
- “终于来了”“爆了”“彻底变天”等情绪化标题，无法识别事件主体。
- 标题只有访谈、评论、观察，且无明确事件。

### 3.3 重复与聚合

排除或合并：

- 同一 URL、同一 canonical_url。
- 标题高度相似且主体一致的转载稿。
- “早报/晚报/8点1氪/每日要闻”类聚合文章：只在无法抓到单条新闻时作为线索，不直接进入日报主体。
- 同一事件多家媒体报道时，日报保留权威来源或信息增量最大的一条，其余作为佐证链接。

## 4. 必须保留的高价值事件

### 4.1 政策监管

保留条件：

- 国务院、发改委、工信部、财政部、央行、证监会、交易所、网信办、市场监管总局等发布的政策、征求意见、处罚、许可、备案、审查。
- 涉及 AI、芯片、数据要素、机器人、新能源车、平台经济、金融监管、IPO 审核、跨境上市、出口管制、关税、反垄断。

示例关键词：

- 政策、监管、征求意见、办法、通知、处罚、审查、备案、牌照、试点、标准、指引、问询、立案、反垄断、出口管制。

### 4.2 公司公告与资本动作

保留条件：

- 上市公司财报、业绩预告、重大合同、定增、回购、并购、分拆、股权激励、减持、控制权变更。
- IPO 受理、问询、过会、注册、发行、上市、终止、港股递表、聆讯、招股。
- 一级市场融资，只有符合下列条件之一才保留：
  - 金额明确且不低于 5000 万元人民币或等值外币。
  - 投资方为知名产业资本、头部 VC/PE、上市公司或国资。
  - 所属方向为本项目六大赛道。
  - 公司已进入商业化、量产、订单、交付阶段。

示例关键词：

- IPO、递表、聆讯、过会、注册、上市、终止、融资、领投、并购、收购、重组、定增、回购、减持、财报、营收、净利润、亏损、订单、合同、量产。

### 4.3 市场与宏观变量

保留条件：

- CPI、PPI、PMI、社融、M2、进出口、工业增加值、固定资产投资、社零、就业、地产、财政、利率、汇率、国债收益率、黄金、原油、美元指数。
- 美联储、欧洲央行、日本央行等主要央行利率决议和重要表态。
- A股、港股、美股重要指数大幅波动，并能说明触发因素。

注意：

- “市场上涨/下跌”本身不够，必须有原因、数据、政策、行业或资金线索。
- 普通行情快讯只进入看板，不一定进入日报正文。

### 4.4 产业链变化

保留条件：

- 涨价、降价、缺货、订单、产能、库存、交付、出口限制、供应商切换、重要客户导入。
- 对半导体、PCB、服务器、光模块、液冷、机器人零部件、AI 算力、消费电子、汽车电子有传导价值。

示例关键词：

- 涨价、报价、库存、产能、扩产、订单、交付、良率、晶圆、封装、先进制程、HBM、GPU、ASIC、服务器、光模块、PCB、铜箔、玻纤、减速器、丝杠、传感器。

## 5. 六大赛道细则

### 5.1 AI 应用 / AI Agent

保留：

- 大模型产品发布、Agent 商业化、企业级 AI 落地、算力采购、模型价格变化、监管政策、AI 基础设施、AI 安全与出口管制。
- 与上市公司、融资、订单、客户、收入、成本、监管、产业链相关的 AI 新闻。

剔除：

- “AI 写作技巧”“Prompt 技巧”“某产品体验”“AI 趣味玩法”。
- 只有 AI 概念但没有商业、资本、政策、产业链影响的文章。
- “AI+体育/娱乐/教育”泛话题，除非有明确财务或监管影响。

### 5.2 半导体 / PCB

保留：

- 芯片、晶圆、封装测试、EDA、IP、光刻、材料、设备、HBM、存储、先进封装、服务器 PCB、AI PCB、产能、报价、订单。
- 国内替代、出口管制、供应链变化、上市公司公告。

剔除：

- 泛科普、商业史、人物访谈，除非含明确订单、财报、政策或技术突破的产业影响。

### 5.3 具身智能与机器人

保留：

- 人形机器人、工业机器人、具身智能模型、关键零部件、量产、订单、融资、上市公司布局。
- 电机、减速器、丝杠、传感器、控制器、执行器、机器人整机交付。

剔除：

- 展会花絮、概念视频、科幻式介绍、没有订单或融资的产品展示。

### 5.4 IPO / A股科技

保留：

- A股、科创板、创业板、北交所 IPO 进展。
- 科技公司上市、并购重组、监管问询、财报、定增、回购、减持。
- 影响科技成长股估值的流动性、政策、行业景气。

剔除：

- 与科技成长方向无关的小微企业融资或地方招商。

### 5.5 港股科技

保留：

- 港股科技平台公司、创新药、AI、半导体、消费电子、互联网、汽车科技的财报、回购、南向资金、监管、估值修复、IPO。
- AH 映射、港股通、恒生科技指数、平台经济政策变化。

剔除：

- 泛港股行情但没有科技权重、资金或政策线索。

### 5.6 全球市场与宏观变量

保留：

- 主要央行、美元、美债、黄金、原油、汇率、海外科技股、半导体出口限制、关税、地缘冲突导致的能源或供应链影响。
- 海外科技巨头财报、CapEx、AI 算力、芯片采购、供应链订单。

剔除：

- 普通国际政治新闻、社会新闻、战争现场报道，除非明确影响油价、汇率、供应链、风险偏好或中国资产。

## 6. 评分模型

每条新闻计算 `relevance_score`，满分 100。建议按以下权重：

| 维度 | 分值 | 说明 |
| --- | ---: | --- |
| 赛道相关性 | 0-30 | 命中六大赛道且不是泛概念 |
| 事件确定性 | 0-20 | 有公告、数据、金额、政策、订单、财报等明确事件 |
| 来源权威性 | 0-15 | 官方/交易所/上市公司/权威财经媒体高分 |
| 市场影响 | 0-15 | 对股价、行业景气、资金、估值、风险偏好有影响 |
| 信息新鲜度 | 0-10 | 24 小时内高分，旧闻低分 |
| 可验证性 | 0-10 | 有原文链接、主体、时间、数据来源 |

### 6.1 分数准入

- `>= 75`：自动纳入日报重点新闻。
- `60-74`：进入候选池，由日报生成时按赛道配额选择。
- `45-59`：只在权威来源、官方数据或同一事件多源验证时保留。
- `< 45`：不进入日报，只可保留在原始抓取日志。

### 6.2 强制加分

- 官方/监管/交易所/上市公司公告：`+15`
- 明确金额、订单、财报数据、政策文件：`+10`
- 命中两个以上核心赛道：`+8`
- 同一事件被两个以上权威来源报道：`+8`
- 与重点上市公司或指数权重股直接相关：`+8`

### 6.3 强制扣分

- 标题情绪化但无明确主体：`-15`
- 生活方式、体育、娱乐、城市文化等弱相关：`-30`
- 聚合早报/晚报类文章：`-20`
- 无原文链接或链接不可访问：`-10`
- 只有观点无事实增量：`-15`

## 7. 来源分级

### 7.1 A 级来源

默认高权重，符合主题即可优先保留：

- 官方机构：国家统计局、央行、证监会、交易所、工信部、发改委等。
- 上市公司公告、交易所公告。
- 权威财经媒体：证券时报、财联社、第一财经、财新、华尔街见闻等。

### 7.2 B 级来源

需要命中赛道和明确事件：

- 36氪、虎嗅、晚点 LatePost、界面、21财经等。
- 此类来源容易混入泛科技、商业故事和生活方式内容，不能全量进入日报。

### 7.3 C 级来源

只做线索，不直接作为结论：

- 社区、论坛、社交媒体、个人博客、自媒体。
- 需要二次验证，除非原文链接指向公告或权威来源。

## 8. RSS 源特殊处理

### 8.1 36氪

问题：

- RSS 中包含大量创业故事、商业观察、消费、人物、早报聚合。

处理：

- 保留 IPO、融资、并购、AI、机器人、半导体、财报、上市公司相关。
- “8点1氪”等聚合文章默认不进入日报，只用于拆分线索。
- 消费品牌、城市商业、职场、教育、体育、文化类默认剔除。

### 8.2 虎嗅

问题：

- RSS 覆盖范围非常宽，社会、文化、观点、生活方式文章很多。

处理：

- 只保留与资本市场、科技产业链、AI 商业化、半导体、机器人、宏观变量、上市公司相关的文章。
- 商业史、人物访谈、社会观察、城市文化、体育、书店、教育、食品安全等默认剔除。
- “观点强、事实弱”的文章进入候选池，不进入重点新闻。

### 8.3 证券时报等权威财经媒体

处理：

- 优先保留政策、监管、公告、市场数据、上市公司、IPO、机构资金。
- 但社会民生、消费维权、非资本市场案件仍应剔除。

### 8.4 国家统计局

处理：

- 最新发布与数据解读均保留，但日报中应合并同一指标的“发布稿”和“解读稿”。
- 标题中包含 CPI、PPI、PMI、工业、社零、投资、房地产、就业、人口、收入、进出口等优先。

## 9. 日报配额建议

日报不建议超过 30 条主体新闻。建议结构：

| 板块 | 数量 |
| --- | ---: |
| 今日最重要新闻 | 5-8 条 |
| 六大赛道分章 | 每章 3-5 条 |
| 市场与宏观变量 | 3-6 条 |
| 风险提示与后续跟踪 | 3-5 条 |
| 原文清单 | 最多 50 条 |

如果某赛道当天低质量新闻很多，不应强行填满；可以写“暂无高质量新增”。

## 10. 标准化字段

每条入库新闻建议增加以下字段：

```json
{
  "is_finance_relevant": true,
  "relevance_score": 82,
  "decision": "include",
  "decision_reason": "命中 AI 应用赛道，包含上市公司订单与金额",
  "matched_tracks": ["AI 应用 / AI Agent"],
  "matched_events": ["公司公告", "订单"],
  "negative_flags": [],
  "source_level": "B",
  "verification_level": "single_source",
  "report_section": "AI 应用 / AI Agent"
}
```

`decision` 可选值：

- `include`：进入日报。
- `candidate`：候选，日报生成时按分数和配额选择。
- `archive_only`：仅保留原始库，不进入日报。
- `exclude`：过滤掉。

## 11. 可直接实现的伪规则

```python
def should_include(article):
    text = f"{article.title} {article.content}".lower()

    if is_duplicate(article):
        return "exclude"

    if is_outdated(article) and not has_market_followup(article):
        return "exclude"

    if match_hard_negative(text) and not has_market_link(text):
        return "exclude"

    score = 0
    score += track_score(text)
    score += event_score(text)
    score += source_score(article.source_id)
    score += market_impact_score(text)
    score += freshness_score(article.publish_time)
    score += verification_score(article.url)

    score += bonus(article)
    score -= penalty(article)

    if score >= 75:
        return "include"
    if score >= 60:
        return "candidate"
    if score >= 45 and article.source_level == "A":
        return "candidate"
    return "archive_only"
```

## 14. 日报生成时的最终检查

生成日报前必须做一次最终检查：

- 是否有标题明显不属于财经科技判断。
- 是否每条都有原文链接、来源、发布时间。
- 是否每条都有进入日报的理由。
- 是否同一事件重复出现超过 2 条。
- 是否“其他”分类过多。如果“其他”超过 20%，说明分类或过滤规则需要调整。
- 是否六大赛道中某一赛道被泛标题污染，例如所有 AI 都被归入 AI 应用。

## 15. 示例判断

| 标题类型 | 判断 | 原因 |
