# 迁移到公司电脑说明

## 今晚在这台电脑上

1. 双击 `pack_project.bat`，或在 PowerShell 里运行 `.\pack_project.ps1`。
2. 打开 `portable_packages` 文件夹。
3. 找到最新生成的 `finance-radar_时间戳.zip`。
4. 把这个 zip 放到 U 盘、网盘，或任何明天公司电脑能拿到的位置。

## 明天在公司电脑上

1. 解压 `finance-radar_时间戳.zip` 到普通目录，例如 `D:\finance-radar`。
2. 确认电脑安装了 Python 3.10 或以上版本。
3. 双击 `run_web.bat`。
4. 浏览器打开 `http://127.0.0.1:5000`。
5. 在左侧勾选来源，点击“开始抓取”，文章明细表里的“原文链接”会跳到网站原文。

## 关于 API Key

打包脚本默认不会打包 `.env`，避免密钥泄露。

如果公司电脑也要使用抓取 AI：

1. 复制 `.env.example`，改名为 `.env`。
2. 在 `.env` 里填入授权使用的 `FETCH_AI_API_KEY`。
3. 如后续要接 GPT/Claude 分析 AI，再填入对应的 `ANALYSIS_GPT_*` 或 `ANALYSIS_CLAUDE_*`。
4. 重新运行 `run_web.bat`。

## 当前关键功能

- 左侧采集范围按来源性质分组。
- 官方公告和官方数据已合并为“官方信息”。
- 支持 36氪、虎嗅、证券时报、国家统计局等来源。
- 文章明细表展示原文标题和原文链接。
- 支持生成抓取计划、导出 CSV、生成 Markdown 日报。
- 使用本地 JSON 存储：`data/local_store.json`。

## 如果公司电脑不能运行

优先检查：

1. 是否安装 Python 3.10+。
2. 是否解压到普通英文路径或短路径，例如 `D:\finance-radar`。
3. 是否有旧进程占用 `5000` 端口。
4. 公司网络是否拦截 RSS 或外部媒体站点。
5. 如果窗口显示乱码命令错误，请确认使用的是最新的 `run_web.bat`。
